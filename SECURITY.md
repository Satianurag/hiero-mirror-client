# Security Policy

## Supported Versions

| Version | Supported |
|---------|-----------|
| 0.2.x   | Yes       |
| < 0.2   | No        |

## Reporting a Vulnerability

If you discover a security vulnerability in this project, please report it responsibly.

**Do not open a public GitHub issue for security vulnerabilities.**

Instead, please report it by emailing the maintainer or by using [GitHub's private vulnerability reporting](https://github.com/Satianurag/hiero-mirror-client/security/advisories/new).

When reporting, please include:

- A description of the vulnerability
- Steps to reproduce the issue
- The potential impact
- Any suggested fixes (if applicable)

## Response Timeline

- **Acknowledgement**: within 48 hours
- **Initial assessment**: within 5 business days
- **Fix or mitigation**: as soon as practical, depending on severity

## Scope

This policy covers the `@satianurag/hiero-mirror-client` npm package and its source code. It does **not** cover the upstream Hedera/Hiero Mirror Node REST API itself — please report those issues to the [Hiero project](https://github.com/hiero-ledger) directly.

## Disclosure

We follow a coordinated disclosure process. We will work with you to understand and address the issue before any public disclosure. Credit will be given to reporters unless they prefer to remain anonymous.
